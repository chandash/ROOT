We found that there are lots of duplicate oplog serial numbers at this site.
So we are going to restore the database from backup and scrub the oplogs.
This should clean up the duplicates.

You can either just simply scrub up to the past 3 months or so.  There appears to be 2-3 million oplogs created since the value of the last duplicate so I am assuming that there were not 3 million oplogs created in the last 3 months.
If you want to be more accurate use the following command to find the date of the most recent duplicate serial number from the ouptut of the command we ran earlier:


select SERIAL_NO, count(*) from RATL_OPLOG_LONG group by SERIAL_NO having count(*) > 1 order SERIAL_NO by asc;.

Use that value in this command:
